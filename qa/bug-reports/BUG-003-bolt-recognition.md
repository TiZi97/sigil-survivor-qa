# BUG-003 — 번개 시길 인식률 저하 (라이브 유저 제보)

| 필드 | 내용 |
|---|---|
| ID | BUG-003 |

| 심각도 | Major (핵심 시스템인 시길 드로잉의 특정 도형이 사실상 절반 확률로 실패 + 게이지 손실 페널티까지 발생) |

| 발생 조건 | 간헐 (유저의 번개 드로잉 스타일에 따라 편차) |
| 재현/정량화 | 체감 제보를 수치로 전환하기 위해 **합성 제스처 리그**(recognition_rig.js) 제작 — 유저 편차(꺾임 높이·되돌림 폭·기울기·손떨림·긋는 속도·상하 방향)를 시드 고정 난수로 재현, 도형별 500회 측정 |
| 기대 결과 | 모든 시길 도형 인식률 90% 이상 |
| 실제 결과(수정 전) | **bolt 43.4%** (500회 중 불발 282) · line 66.8% · vee/circle 100% |

## 원인 분석 (2건 복합)

1. **번개 템플릿 부족**: 원은 시작점 4종 × 방향 2종 = 템플릿 8개로 커버되는 반면,
   번개는 **단일 기하 × 2방향 = 2개**뿐. 유저마다 다른 꺾임 위치·되돌림 폭·좌우
   방향이 단일 템플릿과 매칭되며 점수가 임계값(0.80)에 미달.
2. **점 수 하한(MIN_POINTS=10)의 과차단**: 짧은 도형을 빠르게 그으면 입력 점이
   10개 미만 → **인식을 시도조차 하지 않고 기각**. 직선 불발(33.2%)의 전원인이
   이것으로, 게이트 수치를 바꿔도 결과가 변하지 않는 것에서 역추적해 발견.

## 수정 과정 — 가설 4건 중 2건 채택, 2건 기각 (리그 측정 기준)

| 가설 | bolt | line | 판정 |
|---|---|---|---|
| (수정 전 베이스라인) | 43.4% | 66.8% | — |
| ① 번개 템플릿 파라메트릭 확장 (8기하×2방향) | 73.8% | 66.8% | ✅ 채택 |
| ② 직선도 판정 전 스무딩 | 73.8% | 66.8% | ❌ 기각 — 번개→직선 오인식 120건 유발 (v0.2에서 잡은 버그 유형의 회귀) |
| ③ 직선 게이트 완화 0.88→0.82 | 94.8% | 99.8%* | ❌ 기각 — 대조 실험 결과 ④만으로 동일 효과 + 오인식 9건→1건 |
| ④ MIN_POINTS 10→6 + 최소 경로 길이 25px 가드 | **96.4%** | **99.8%** | ✅ 채택 |

*③의 line 개선처럼 보인 것은 실제로는 ④의 효과였음 — 변인 분리 대조 실험으로 확인.

수정: v0.9.2. 회귀 방지: 리그를 스킬 정식 도구(`recognition_rig.js`)로 편입 —
목표 미달 시 exit 1을 반환해 인식률 회귀를 자동 검출 (TC "인식률 자동 검증" AUTO 등록).

**QA 시사점**: ①"발동이 안 된다"는 체감 제보는 측정 도구로 정량화한 뒤 고쳐야
한다 — 감으로 고친 가설 2건은 측정에서 기각됐고, 그중 하나는 과거 버그를
재발시킬 뻔했다. ②증상(번개 불발)의 원인이 하나라고 가정하지 말 것 — 실제로는
독립된 원인 2건의 복합이었고, 두 번째 원인은 게이트 수치 변경에 결과가 반응하지
않는 이상 신호에서 역추적됐다.
