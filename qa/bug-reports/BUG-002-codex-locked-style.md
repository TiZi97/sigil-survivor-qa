# BUG-002 — 시길 도감에서 미발견 시길이 발견된 것처럼 표시됨

| 필드 | 내용 |
|---|---|
| ID | BUG-002 |
| 제목 | [UI/도감] 미발견 시길 슬롯에 발견 상태 스타일이 적용됨 |
| 심각도 | Minor (기능 동작에는 문제없으나 신규 유저에게 잘못된 상태 정보 전달 + 히든 위치 정보 누출) |
| 우선순위 | P2 |
| 환경 | v0.9 (GitHub Pages 라이브) · Chrome/Windows |
| 발견 경로 | **배포 후 실플레이 중 발견** — 클리어 기록이 없는 신규 세션에서 확인 |
| 발생 조건 | 항상 (시길을 하나도 발견하지 않은 상태에서 도감 진입 시) |
| 재현 스텝 | 1) 클리어/발견 기록 없는 새 브라우저 프로필로 접속 2) 로비에서 "시길 도감" 클릭 3) 첫 번째 줄 슬롯 4칸 관찰 |
| 기대 결과 | 발견한 시길 0/20이므로 20칸 전부 동일한 미발견(?) 상태로 표시 |
| 실제 결과 | 슬롯 0~3번(첫 줄 4칸)만 초록빛 배경이 채워져 발견된 것처럼 보임 |
| 원인 분석 | `renderCodex()`가 미발견 분기에서도 시길이 정의된 슬롯에 티어 클래스(`t1`)를 부여 → CSS `.slot.t1`의 초록 계열 배경이 적용됨. 발견 여부(`codex[key]`)와 무관하게 **데이터 존재 여부**로 스타일이 갈린 것이 원인. 부수 문제로, 어느 슬롯에 실제 시길이 있는지가 시각적으로 노출되어 히든 발견 설계의 정보 무결성도 훼손됨 (v0.5에서 "발견한 시길만 강화 카드 등장"으로 지켰던 원칙과 상충). |
| 수정 및 검증 | v0.9.1에서 미발견 분기의 티어 클래스 부여 제거 — 미발견 슬롯은 시길 존재 여부와 무관하게 동일하게 표시. 신규 세션에서 20칸 균일 표시 확인, 발견 후 해당 슬롯만 점등되는 기존 동작 회귀 확인. 회귀 TC 등록: "도감 미발견 슬롯 균일 표시" |

**QA 시사점**: 상태 표시 UI는 "데이터가 존재하는가"가 아니라 "사용자가 도달했는가"를 기준으로 렌더링해야 한다. 이 버그는 로컬 개발 중에는 개발자 브라우저에 발견 기록이 쌓여 있어 드러나지 않았고, **클리어 기록이 없는 깨끗한 환경(신규 유저 관점)에서만 재현**됐다 — 신규 프로필 테스트가 회귀 스위트에 필요한 이유. (test-plan의 난이도 해금 TC가 새 프로필을 전제하는 것과 같은 맥락)

---
*연관 개선: 같은 패치에서 미발견 슬롯 클릭 시 문양 힌트(그려야 할 궤적만 표시, 이름·효과는 비공개)를 추가 — 접근성을 높이되 히든 발견 재미는 유지하는 절충안.*
