# Sigil Survivor — 개발·QA 로그 (v0.1 → v0.8)

> 1인 프로젝트로 게임 개발과 QA를 병행하며, 매 버전 "만들고 → 검증하고 →
> 회귀 항목을 누적하는" 사이클을 반복했다. 이 문서는 그 과정의 기록이다.

## 버전 히스토리

### v0.1 — 도형 프로토타입
- 단일 HTML 파일. 이동(WASD)/대시(무적 프레임, 쿨다운 2.5초), 무기 3종
  (유도 화살·궤도 오브·부채꼴 참격), 패시브 4종, 적 3종(추격자/질주자/거수),
  레벨업 3택. 그래픽은 전부 도형으로 렌더링.
- 설계 결정: 점프 제거(탑다운 명료성 우선), **모든 밸런스 수치를 데이터
  테이블로 분리** — 이후 모든 QA 활동의 기반이 된 구조 결정.

### 인식 테스트 리그 (병행 트랙) — 게임 밖에서 먼저 검증
게임의 핵심 차별점인 "마우스로 도형을 그려 스킬 발동"($1 Recognizer 기반)은
게임에 바로 넣지 않고 **전용 테스트 리그를 별도로 만들어 인식률부터 측정**했다.
혼동 행렬, 도형별 인식률 집계, CSV 내보내기 포함.

- **리그 v0.2 — 첫 실전 버그**: 얕은 V/빠른 번개가 '직선'으로 오인식되는 문제를
  혼동 행렬에서 발견 → 재현 조건 특정("얕게 그리면") → 원인 진단(1D 납작도
  판정 임계값 0.22가 과도) → 0.10으로 조정 + 직선도 게이트(≥0.88) 도입 →
  오인식 케이스와 정상 직선 양쪽 회귀 확인. **프로젝트 버그 리포트 1호.**
- **리그 v0.3 — Tier 2 확장**: 도형 10종으로 확장하며 원↔나선↔무한 충돌
  리스크를 회전량 게이트(부호 있는 총 회전각: 원≈360°, 나선≈720°, 무한≈0°)로
  해소. "게임에 넣기 전에 리그에서 측정 후 통합"을 표준 프로세스로 확립.

### v0.3 — 시길 시스템 통합

### v0.4 — 난이도 & 보스

  (×1.0/1.5/2.0). 보스 '처형자'(돌진 예고 패턴, 부하 소환) — 난이도별 3/5/6분 등장.

  순서 의존성을 경험.

### v0.9 — 자석 아이템 · 마력 오라 · 성능 최적화

  드로잉 시길로 검토했으나(U자는 V와 오인식 리스크) 장르 관례에 맞는 드랍
  아이템으로 전환 — 인식기를 건드리지 않아 회귀 범위 최소화.
- **수호 오브 → 마력 오라 교체**: 저성능 무기 리워크. 상시 발동 지속 피해장
  (Lv1 반경 80/DPS 10 → Lv5 반경 144/DPS 38.2) + 범위 내 감속(10→18%).
  성장 정체성은 반경 확장. 신규 무기 타입 추가에 따라 검증 도구의 DPS 공식과

  ③젬 렌더 save/rotate 제거 ④피격 플래시 cx.filter(고비용) → 오버레이 전환
  ⑤근접 젬 드랍 병합(30px)으로 필드 젬 수 억제.

### v0.9.4 — 신규 시길 2종: 회생 ❤ · 성광 ⭐ (내부 피드백 반영)

  하트는 첫 측정부터 100%(원과의 혼동 우려는 기우). 별은 54.6%로 낙제 —
  원인은 오각별의 **시작 꼭짓점에 따른 경로 위상 차이**($1은 경로 순서 비교).
  시작점 5종 전부를 템플릿화(4→10종)해 99.2% 달성. 상호 오인식 0 유지.
- **회생(❤)**: HP 100% 회복, 비용 85(게이지의 85% — 비상 버튼 포지션).
  성장은 회복량이 아닌 **초과 회복분의 보호막 전환**(+10%p/Lv) — Lv1부터
  완전한 효과를 주되 레벨업 가치를 별도 축으로 설계.
- **성광(⭐)**: 6~12초 공격력 +25→53%/이속 +15→27%. 파워업 체감 전달용

## 이 로그에서 보이길 바라는 것

1. **검증을 앞당기는 습관** — 리스크가 큰 시스템(제스처 인식)은 게임에 넣기
   전에 전용 리그에서 측정했다. 버그를 본편에서 만나면 비용이 커진다.
2. **버그 → 회귀의 순환** — 발견된 버그(오인식, 조종사 소실)는 수정으로
   끝나지 않고 회귀 케이스로 남아 다음 버전에서 재검증됐다.
3. **수치로 말하기** — 밸런스 변경, 성능, 인식률 전부 측정값을 근거로 결정했다.
