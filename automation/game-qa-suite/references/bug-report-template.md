# 버그 리포트 템플릿

버그 리포트는 아래 8개 필드를 채운다. 핵심 원칙:
- **재현 스텝은 처음 보는 사람이 그대로 따라 할 수 있게** (전제 지식 가정 금지)
- **증상과 원인을 분리** — 원인 분석은 확인된 범위까지만, 추정은 추정이라 명시
- 심각도(Severity)는 영향 범위, 우선순위(Priority)는 수정 순서 — 별개 축으로 판단

```
ID:          BUG-###
제목:        [발생 영역] 한 줄 증상 요약
심각도:      Critical / Major / Minor / Trivial
우선순위:    P0 / P1 / P2 / P3
환경:        빌드 버전 · 브라우저/OS · 해상도
발생 조건:   (항상 / 간헐 n회 중 m회 / 특정 조건)
재현 스텝:   1) ... 2) ... 3) ...
기대 결과:   ...
실제 결과:   ... (스크린샷/영상 첨부)
원인 분석:   (확인된 경우만)
수정 및 검증: (수정 내역 + 회귀 TC ID)
```

---

## 실제 사례: BUG-001 — 피격 시 플레이어 조종사 리소스 소실

| 필드 | 내용 |
|---|---|
| ID | BUG-001 |
| 제목 | [렌더링] HP 50% 이하 피격 시 UFO 탑승 캐릭터가 사라짐 |
| 심각도 | Major (핵심 캐릭터 시인성 훼손, 진행은 가능) |
| 우선순위 | P1 |
| 환경 | v0.7 · Chrome/Windows · 1920×1080 |
| 발생 조건 | 항상 (HP가 50% 임계값을 하향 통과하는 즉시) |
| 재현 스텝 | 1) 보통 난이도 새 게임 2) 적에게 고의 피격으로 HP를 49 이하로 감소 3) 플레이어 스프라이트 관찰 |
| 기대 결과 | 기체 파손 상태로 바뀌되 조종사는 유지 |
| 실제 결과 | 조종사가 사라지고 빈 기체만 표시됨 (스크린샷 첨부) |
| 원인 분석 | 정상 스프라이트(player.png)는 124×145로 조종사 포함, 파손 스프라이트(player_d1/d2.png)는 124×90 **기체 단독** 리소스. HP 비율 50%/25% 분기에서 이미지 교체 시 조종사 미포함 에셋으로 전환됨. 코드 결함이 아닌 **에셋 규격 불일치**. |
| 수정 및 검증 | v0.8에서 파손 스프라이트에 조종사 합성(124×145 규격 통일, 이음새 8px 알파 페이드). 렌더링이 이미지 비율 기반이라 코드 무수정. 회귀 TC: "플레이어 파손 스프라이트 조종사 유지" |

**QA 시사점**: 동일 오브젝트의 상태별 에셋은 규격(캔버스 크기·앵커) 일치를 입고 시점에 검증해야 한다. → 에셋 입고 체크리스트에 "상태 변형 에셋 규격 일치" 항목 추가.
