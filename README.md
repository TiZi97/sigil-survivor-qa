## QA 파이프라인 — "코드가 곧 명세"

이 게임은 모든 밸런스 수치가 데이터 테이블에 모여 있는 데이터 주도 구조입니다.
그 테이블을 명세(single source of truth)로 삼아:

```
게임 소스 ─① 스펙 추출→ spec.json ─② 불변식 검증 18건→ 밸런스 리포트 (FAIL 시 exit 1)
                          └─③ TC 자동 생성→ testcases.csv (수동 회귀 목록과 병합)
빌드 ─④ Playwright 스모크 5종 (~20초)→ 테스트 가능 빌드 판정
```

운영 규칙 두 가지를 지켰습니다:
- **버그 1건 수정 = 회귀 TC 1건 추가** — 수정으로 끝내지 않고 재발 방지를 스위트에 누적
- **검증 도구도 검증한다** — 의도적으로 틀린 기준을 넣어 FAIL이 뜨는지 확인(네거티브 테스트)

파이프라인 스크립트(①~③)는 외부 전송 없이 로컬에서 도는 파이썬이라,
기밀 데이터를 다루는 실무 환경에도 그대로 이식 가능한 구조입니다.

## AI 협업에 대해

게임 구현과 스크립트 작성은 Claude와 협업했습니다. 테스트 전략 수립,
무엇을 검증할지 정의, 밸런스 목표 설정, 버그 판정과 최종 검토는 직접 수행했으며,
단발성 생성이 아니라 재사용 가능한 QA 파이프라인(Claude Skill)으로
도구화하는 설계에 중점을 뒀습니다. AI 산출물 역시 QA 대상이라는 원칙으로
검증 도구에 대한 네거티브 테스트를 함께 수행했습니다.
